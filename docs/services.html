Design

The core business functions are develop in services either using microservices or event driven actors. The Platform provides the infrastructure for a fast and portable implementation. Start by defining your software architecture - or not but make a concious decision about this. The platform inceases your speed by making some basic architectural choices for you and providing tools to help you. Next select the correct boostrap toolkit and start with the software design.

Sofware Design:

The software design includes how a component is implemented, the interface and api, sequence of actions and exception handling.

  1. Start by designing the API
  2. Map out the business process(activities) & sequences of actions with
  3. Next model the state - data architecture of the components e.g. the service or the application (redux & views).

Design the API / Events in Swagger

First we start designing the API with Swagger. The best tool to do this is the swagger editor. I suggest adding the swagger editor to the docker-compose file in your project:

services:
  swaggereditor:
    image: swaggerapi/swagger-editor
    ports:
      - "80:8080"
